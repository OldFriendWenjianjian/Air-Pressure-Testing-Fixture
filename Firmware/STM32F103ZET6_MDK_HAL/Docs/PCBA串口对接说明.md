# 气压检测工装 PCBA 串口对接说明

本文档摘自并细化 `气压检测工装功能逻辑表.md` 中 `4. 状态功能逻辑表` 和 `4.1 串口协议规则`，用于 PCBA 软件与 STM32F103ZET6 主控串口对接。

## 1. 通信基本规则

- MCU 为主机，8 路 PCBA 为从机。任何时候只能由主机主动发起，从机只能在收到指令后立即返回。
- `UART1` 至 `UART8` 对应 PCBA 通道 1 至通道 8。
- 串口参数固定为 `115200 8N1`，即 115200 baud、8 数据位、无校验、1 停止位。
- MCU 采用轮询方式依次向通道 1 至通道 8 发送命令。
- 常规命令回包等待时间为 `10ms`。开机压合后的第一条唤醒命令 `00` 等待时间为 `2s`。
- 气压值统一使用 4 字节小端无符号整数，单位为 `0.001mmHg`。例如 `100mmHg` 表示为十进制 `100000`。

## 2. 帧格式

| 字段 | 字节数 | 说明 |
| --- | ---: | --- |
| `55 AA` | 2 | 固定帧头 |
| `CMD` | 1 | 命令字 |
| `CH` | 1 | 通道号，`01` 至 `08` |
| `LEN_L LEN_H` | 2 | `DATA` 长度，小端 |
| `DATA` | N | 参数或返回数据 |
| `CRC_L CRC_H` | 2 | `CRC16/MODBUS`，小端 |

CRC 校验范围为 `CMD + CH + LEN_L + LEN_H + DATA`，不包含帧头 `55 AA`。CRC 初值 `0xFFFF`，多项式 `0xA001`。

## 3. 命令字

| 命令 | 名称 | 方向 | DATA | PCBA 返回 |
| --- | --- | --- | --- | --- |
| `00` | 唤醒机器 | MCU -> PCBA | 无 | 空应答包 `7F` |
| `01` | 进入测试模式 | MCU -> PCBA | 无 | 空应答包 `7F` |
| `02` | 记录零点 | MCU -> PCBA | 无 | 空应答包 `7F` |
| `03` | 开机 | MCU -> PCBA | 无 | 空应答包 `7F` |
| `04` | 查询低电状态 | MCU -> PCBA | 无 | `7F` 带 1 字节数据 |
| `05` | 查询正常状态 | MCU -> PCBA | 无 | `7F` 带 1 字节数据 |
| `10` | 标定压力同步 | MCU -> PCBA | 4 字节压力值 | 空应答包 `7F` |
| `11` | 测试压力查询 | MCU -> PCBA | 无 | `11` 带 4 字节 PCBA 测量值 |

查询低电状态 `04`：PCBA 返回数据 `00` 表示低电，`01` 表示非低电。主控在 PCBA 供电切到 4.5V 并等待 0.5s 后查询，期望返回 `00`。

查询正常状态 `05`：PCBA 返回数据 `01` 表示低电，`00` 表示非低电。主控在 PCBA 供电切回 5V 并等待 0.5s 后查询，期望返回 `00`。

## 4. 阶段串口交互

| 阶段 | MCU 主控发送 | PCBA 返回 | 时序/备注 |
| --- | --- | --- | --- |
| U 盘维护模式 | 不发送 PCBA 业务命令 | 无 | 开机长按 KEY1(PC3) 进入，USB 以 MSC 磁盘方式暴露 W25Q128 |
| 初始化、自动气密、Ready 等待 | 不发送 PCBA 业务命令 | 无 | 主控只进行气罐闭环、保压、等待压合 |
| 压合后第 1 步 | `00` 唤醒机器 | 空应答包 `7F` | 单路等待超时 `2s`，收到后该通道显示 PCBA 在线 |
| 压合后第 2 步 | `01` 进入测试模式 | 空应答包 `7F` | 常规回包等待 `10ms` |
| 压合后第 3 步 | `03` 开机 | 空应答包 `7F` | 主控随后等待 `2s` |
| 压合后第 4 步 | `02` 记录零点 | 空应答包 `7F` | 气压未上压前执行 |
| 压合后第 5 步 | `04` 查询低电状态 | `7F` + 1 字节数据 | 查询前主控将 PCBA 供电切到 4.5V 并等待 0.5s，期望数据 `00` |
| 压合后第 6 步 | `05` 查询正常状态 | `7F` + 1 字节数据 | 查询前主控将 PCBA 供电切回 5V 并等待 0.5s，期望数据 `00` |
| 50mmHg 标定 | `10` + 当前真实压力值 | 空应答包 `7F` | 压力稳定后对 8 路轮询发送 |
| 150mmHg 标定 | `10` + 当前真实压力值 | 空应答包 `7F` | 压力稳定后对 8 路轮询发送 |
| 250mmHg 标定 | `10` + 当前真实压力值 | 空应答包 `7F` | 压力稳定后对 8 路轮询发送 |
| 100mmHg 测试 | `11` 无 DATA | `11` + 4 字节测量值 | 压力稳定后查询 PCBA 当前真实测量值 |
| 200mmHg 测试 | `11` 无 DATA | `11` + 4 字节测量值 | 压力稳定后查询 PCBA 当前真实测量值 |
| 300mmHg 测试 | `11` 无 DATA | `11` + 4 字节测量值 | 压力稳定后查询 PCBA 当前真实测量值 |
| 结果显示/补气 | 不发送 PCBA 业务命令 | 无 | 主控汇总结果并准备下一批次 |

## 5. 典型帧示例

以下示例中的 `CH` 为通道号 `01` 至 `08`，`CRC_L CRC_H` 由 PCBA 或主控按 CRC16/MODBUS 实际计算。

| 场景 | 帧 |
| --- | --- |
| 唤醒机器 | `55 AA 00 CH 00 00 CRC_L CRC_H` |
| 进入测试模式 | `55 AA 01 CH 00 00 CRC_L CRC_H` |
| 开机 | `55 AA 03 CH 00 00 CRC_L CRC_H` |
| 记录零点 | `55 AA 02 CH 00 00 CRC_L CRC_H` |
| 查询低电状态 | `55 AA 04 CH 00 00 CRC_L CRC_H` |
| 查询正常状态 | `55 AA 05 CH 00 00 CRC_L CRC_H` |
| 空应答包 | `55 AA 7F CH 00 00 CRC_L CRC_H` |
| 一字节 YES 应答 | `55 AA 7F CH 01 00 00 CRC_L CRC_H` |
| 一字节 NO 应答 | `55 AA 7F CH 01 00 01 CRC_L CRC_H` |
| 标定压力同步 | `55 AA 10 CH 04 00 BYTE0(LSB) BYTE1 BYTE2 BYTE3(MSB) CRC_L CRC_H` |
| 测试压力查询 | `55 AA 11 CH 00 00 CRC_L CRC_H` |
| 测试压力返回 | `55 AA 11 CH 04 00 BYTE0(LSB) BYTE1 BYTE2 BYTE3(MSB) CRC_L CRC_H` |

## 6. PCBA 侧实现要点

1. PCBA 仅在收到主控完整且 CRC 正确的帧后回包。
2. `CH` 需要与对应 UART 通道匹配；PCBA 回包保持相同 `CH`。
3. `00/01/02/03/10` 成功执行后返回空应答包。
4. `04/05` 返回 `7F` 应答包，`LEN=01 00`，`DATA` 按各自语义填写。
5. `11` 收到后返回命令字 `11`，`LEN=04 00`，`DATA` 为 PCBA 当前压力测量值，单位 `0.001mmHg`，小端。
6. 未知压力、待实测压力、或需要由 PCBA 自身采集后填写的数据，在联调记录中可先用 `?` 标注。
